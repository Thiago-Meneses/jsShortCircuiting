Short Circuiting

Short Circuiting is a behaviour that is built into JavaScript whenever you're evaluating anything

Short circuiting allows things to become simpler.

Short circuiting allows us to use Logical Operators, which can be either && or ||

We use these operators in the same way if we were to compare values in conditional statements, like if statements.

It's also important to use ; at the end of our code lines.

This prevents ASI (Automatic Semicolon Insertion), which can cause erros in our code.

Short circuing can be done surrounded by parenthesis ( ) or curly braces { }

Examples

What follows is a list of examples showing the behaviours and differences of the operatosrs.

1º Example:

                                
let a = true;
let b = false;
    
//Short Circuiting with Parentheses
( a && b );
( a || b );

//Short Circuiting with Curly Braces
{ a && b };
{ a || b };

Console:

                                
false
true

false
true

2º Example:

Using Short Circuiting inside an if statement

                                
var a = true
var b = false

if ( a && b ) {
    console.log("yes");
} else {
    console.log("no");
}

Console:

                                
no

3º Example:

Using Objects and Variables

                                
let myFavoriteNumber = 88
let personObj = { 
    name: "John",
    age: 42
}

if ( personObj.name==="John" && myFavoriteNumber===25 ) {
    console.log("Yes")
} else {
    console.log("No")
}

Console:

                                
No

4º Example:

Using a Ternary Operator

                                
let myFavoriteNumber = 88
let personObj = { 
    name: "John",
    age: 42
}

( personObj.name==="John" && myFavoriteNumber===25 ) ? console.log("Yes") : console.log("No")

Console:

                                
No

Falsie Values

Falsey values are applicable with Short Circuiting

All falsey values are applicable

Example: 
                                        
let myNum = 0
let myName = "Michael"

(myName==="Michael" && myNum) ? console.log("yes") : console.log("No")

Console:

                                        
No

myNum is a falsey value, and is considered as false

Functions without a return statement

When using functions we have to keep in mind that they will always return something, even if we don't manually define a return value

A function that has not return statement will return undefined by default.

Remember that undefined is a falsie value.

Because of this, when we call functions in Short Circuiting, this falsie value will be considered.

Example: 
                                            
function myFunc() {
    let name = "John"
}
let age = 32

( age && myFunc() ) ? console.log("Yes") : console.log("No")

Console:

                                            
No

"No" will be logged into the console

This is because we didn't define a return value for myFunc, and when we call it, undefined is returned as default.

Calling vs Using a Function

Another important thing to understand is that there is a difference between calling a function and using a function.

To call a function we write parenthesis in front of it, or else it will not be called.

Let's use the previous example again, this time without calling the function to see the difference.

Example: 
                                            
function myFunc() {
    let name = "John"
}
let age = 32

(age && myFunc)? console.log("Yes") : console.log("No")

Console:

                                            
Yes

This time "Yes" was logged into the console, and this is because inside of our Short Circuit, we only checked if both "age" and "myFunc" were true, and as both of them are not falsey values, we got "Yes" as a return

Even if myFunc returned undefined as default, we didn't actually call it, we just want to know if it existed, and yes, it does exist
